So, you would have thought that 8 GB of disk space for your virtual machine would be enough to do a bit of .NET development (i.e. install Windows XP and Visual Studio 2008). Unfortunately, I was wrong and had to make my virtualised partition bigger. Making the virtual disk bigger is very straight forward. Resizing the actual (primary active) Windows partition, however, is a bit less trivial.

Here's what I did:
  1. Resize the VMware disk file;
    vmware-vdiskmanager -x 12GB /path/to/file.vmdk
  2. Get knoppix iso (or other bootable Linux iso);
  3. Use iso for your virtual machine's CD-ROM;
  4. Boot your virtual machine (i.e. boot using knoppix); *
  5. Use qtparted (or any other suitable tool on your chosen Linux distro) to resize the bursting partition.
    sudo qtparted

* I had trouble hitting ESC fast enough to get into the boot menu and therefore had to tell VMware to slow down a bit:
echo 'bios.bootDelay="5000"' >> /path/to/file.vmx
